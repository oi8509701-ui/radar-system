# AERIS-10N Phased Array Radar - Проект PLFM_RADAR_NEWS

## 📋 Обзор проекта

**Дорожная карта всех работ:** [`ROADMAP_RADAR.md`](ROADMAP_RADAR.md) (тайлы, синхро-плата, PowerBoard, MainBoard, верификация).

**AERIS-10** — это 10.5 ГГц open-source PLFM phased-array radar с двумя конфигурациями:

### Конфигурации:

| Конфигурация | Тайлов | Элементов | Каналов | Gain | TX Power |
|--------------|--------|-----------|---------|------|----------|
| **AERIS-10N (Nexus)** | 1 | 16 (4×4) | 4 | ~18 dBi | ~1W |
| **AERIS-10E/X (Extended)** | 16 | 256 (16×16) | 64 | ~30 dBi | ~160W |

### Дальность обнаружения (по целям):

| Цель | RCS (σ) | AERIS-10N | AERIS-10E/X |
|------|---------|-----------|-------------|
| **Слон** 🐘 | 100 m² | ~3 км | ~20 км |
| **Автомобиль** 🚗 | 10 m² | ~1.7 км | ~11 км |
| **Человек** 🚶 | 1 m² | ~1 км | ~6.5 км |
| **Дрон** 🚁 | 0.01 m² | ~300 м | ~2 км |
| **Птица** 🐦 | 0.001 m² | ~170 м | ~1.1 км |

**Примечание:** Дальность рассчитана по формуле радара: Rmax = ⁴√[(Pt × Gt × Gr × λ² × σ) / ((4π)³ × Smin)]

### Ключевые компоненты:
- **FPGA**: XC7A50T/XC7A100T (Artix-7)
- **Clock**: AD9523-1 (low-jitter clock distribution)
- **Synthesizers**: 2× ADF4382 (RX/TX LO)
- **Beamforming**: 4× ADAR1000
- **Front-end**: 16× ADTR1107
- **Mixer**: LTC5552
- **MCU**: STM32F746xx (control, sequencing, sensors)
- **PA**: QPA2962 GaN amplifiers (10W, для Extended версии)

### Инструменты MCP

|------------|----------|
| `calculate_patch_antenna` | Расчёт размеров патч-антенны |

### Версии генераторов

#### 4×4 Antenna (4 канала):
- **v21.0**: T1 точно посередине Y=0.0, все пути равны
- **v22.0**: 45° углы, L_QW/2 сегменты
- **v23.0**: Дуги вместо углов, L_QW/4 сегменты
- **v24.0**: Идеальная RF-версия с дугами R=2.5мм

#### 8×16 Antenna (16 каналов):
- **v5.0**: 16 независимых каналов, строка = канал
- **v6.0**: Вертикальные каналы (колонка = канал)
- **v7.0**: Симметричное дерево 1→2→4
- **v8.0**: Абсолютно симметричное дерево
- **v9.0**: T1 точно посередине

### Расчётные формулы

```
Ширина патча: W = c/(2f) × √(2/(εr+1))
Длина патча: L = c/(2f√εeff) - 2ΔL
λ/4 трансформатор: L_QW/4 = λ₀/(4√εeff)
```

Для **RO4350B (εr=3.48, h=0.762мм)** @ **10.5 GHz**:
- **L_QW/4** = 1.105 мм
- **W50** = 1.63 мм
- **W70** = 0.95 мм
- **W100** = 0.55 мм

## 📡 Антенные подсистемы

### Patch 4×4 (Тестовая)
- **Элементов**: 16 (4×4)
- **Каналов**: 4 независимых
- **Размер**: 70×70 мм
- **Назначение**: Отладка RF-тракта, калибровка

### Patch 8×16 (Основная для AERIS-10N)
- **Элементов**: 128 (8×16)
- **Каналов**: 16 независимых
- **Размер**: 130×190 мм
- **Gain**: ~20-22 dBi
- **Назначение**: Основная антенна радара

### Waveguide 32×16 (Для AERIS-10E)
- **Элементов**: 512 (32×16)
- **Тип**: Dielectric-filled slotted waveguide
- **Gain**: ~30-33 dBi
- **Назначение**: Антенна для расширенной версии

### Fractal 32×16 (H-дерево для волноводной решётки)
- **Структура**: Фрактальное H-дерево (рекурсивное деление)
- **Итерации**:
  - **Итерация 0**: Базовый элемент — 1 канал
  - **Итерация 1**: 4 канала (4 ответвления)
  - **Итерация 2**: 16 каналов
  - **Итерация 3**: 64 канала
  - **Итерация 4**: 256 каналов (полное дерево для 32×16)
- **Принцип**: Каждое ответвление заменяется на уменьшенную копию "H"
- **Преимущества**:
  - ✅ Абсолютно равные длины путей до всех элементов
  - ✅ Идеальная симметрия фазы и амплитуды
  - ✅ Минимальные потери в делителях мощности
  - ✅ Самоподобие структуры (масштабируемость)
- **Файл**: [`Antennas/Fractal_32x16/09_47_11.jpg`](Antennas/Fractal_32x16/09_47_11.jpg) — визуализация итераций 0-2

### H-Fractal Antenna — Правила проектирования (по 008.png)

**Геометрия H-фрактала (Stage 7):**

```
         L7 L6 L5 L4 L3 L2 L1 L0  ← Уровни итераций
         ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓
    ┌────┴──┴──┴──┴──┴──┴──┴──┴────┐
    │  H   H   H   H   H   H   H   H │  ← Элементы
    │  │   │   │   │   │   │   │   │ │
    │  H───H───H───H───H───H───H───H │
    │  │                           │ │
    └──┼───────────────────────────┼──┘
       │                           │
    ┌──┴───────────────────────────┴──┐
    │         Ground Plane            │  ← Wg, Lg
    └─────────────────────────────────┘
                 │
              W0 │ ← Feed line (50Ω)
```

**Размеры и их влияние:**

| Параметр | Описание | Влияние |
|----------|----------|---------|
| **L0** | Базовая длина H (уровень 0) | Определяет общий размер антенны |
| **L1, L3, L5, L7** | Длины H на нечётных уровнях | Масштабирование 1/√2 на уровень |
| **L2, L4, L6** | Длины H на чётных уровнях | Масштабирование 1/√2 на уровень |
| **W0** | Ширина линии питания (центр) | Импеданс 50Ω (обычно ~1.63мм для RO4350B) |
| **W1** | Ширина линий H-элементов | Импеданс ветвей (70-100Ω для делителей) |
| **Wg** | Ширина ground plane | Влияет на излучение назад, типично λ/2 |
| **Lg** | Длина ground plane | Должна перекрывать активную область |
| **H** | Толщина подложки | Влияет на εeff, полосу пропускания |

**Правила создания H-фрактальной антенны:**

1. **Масштабирование уровней:**
   ```
   L(n+1) = Ln / √2
   ```
   Каждый следующий уровень уменьшается в √2 раз

2. **Симметрия:**
   - Левая и правая ветви зеркальны
   - Все пути от центра до терминалов равны
   - Фазовый баланс критичен для beamforming

3. **Импеданс:**
   ```
   W0 = 50Ω (вход)
   W1 = 70.7Ω (λ/4 трансформатор) или 100Ω (для делителя 2:1)
   ```

4. **Количество элементов:**
   ```
   N(elements) = 4^n, где n = номер итерации
   Iteration 0: 4 элемента
   Iteration 1: 16 элементов
   Iteration 2: 64 элемента
   Iteration 3: 256 элементов
   ```

5. **Ground plane:**
   - Располагается снизу (B.Cu)
   - Via stitching по периметру
   - Размер > активной области на 10-20%

## 📐 Расчёт микрополосковых патч-антенн (Microstrip Patch Antenna)

### Структура микрополосковой антенны:

```
    ┌─────────────────────────┐  ← Conducting Patch (W × L)
    │    Dielectric Substrate │  ← h (толщина)
    └─────────────────────────┘  ← Ground Plane (Wg × Lg)
```

### Входные параметры для расчёта:

| Параметр | Обозначение | Для 10.5 GHz (RO4350B) |
|----------|-------------|------------------------|
| **Центральная частота** | f₀ | 10.5 GHz |
| **Диэлектрическая проницаемость** | εr | 3.48 (RO4350B) |
| **Толщина подложки** | h | 0.762 мм (30 mil) |
| **Скорость света** | c | 299,792,458 м/с |

---

### 1️⃣ Ширина патча (W)

**Формула:**
```
        c              1
W = ─────── × √─────────────────
       2f₀      (εr + 1) / 2
```

**Для 10.5 GHz на RO4350B:**
```
W = 28.55 / 2 × √(2 / 4.48) = 9.54 мм
```

---

### 2️⃣ Эффективная диэлектрическая проницаемость (εeff)

**Формула:**
```
              εr + 1       εr - 1           1
εeff = ────────────── + ────────── × ──────────────────
                 2            2      √(1 + 12h/W)
```

**Для 10.5 GHz на RO4350B:**
```
εeff = 2.24 + 1.24 × 0.435 = 3.08
```

---

### 3️⃣ Удлинение длины (ΔL)

**Формула (Hammerstad):**
```
      (εeff + 0.3) × (W/h + 0.264)
ΔL = ────────────────────────────────── × h
      (εeff - 0.258) × (W/h + 0.8)
```

**Для 10.5 GHz на RO4350B:**
```
ΔL = 0.412 × 0.762 × 1.089 = 0.342 мм
```

---

### 4️⃣ Длина патча (L)

**Формула:**
```
       c                c
L = ─────────────────────── = ───────────── - 2ΔL
       2f₀ × √εeff       2f₀ × √εeff
```

**Для 10.5 GHz на RO4350B:**
```
L = 28.55 / (2 × √3.08) - 2×0.342 = 8.13 - 0.684 = 7.35 мм
```

---

### 5️⃣ Размеры заземления (Wg, Lg)

**Рекомендации:**
```
Wg = W + 6h = 9.54 + 6×0.762 = 14.11 мм
Lg = L + 6h = 7.35 + 6×0.762 = 11.92 мм
```

**Для массива 4×4:**
```
Wg_total ≈ 4×W + 3×spacing + margins
Lg_total ≈ 4×L + 3×spacing + margins
```

---

### 6️⃣ Длины линий передачи

**λ/4 трансформатор:**
```
L(λ/4) = λ₀ / (4 × √εeff) = 28.55 / (4 × √3.08) = 4.06 мм
```

**Ширины линий для 50/70/100Ω (RO4350B, h=0.762мм):**

| Импеданс | Ширина линии |
|----------|--------------|
| **50Ω** | W50 = 1.63 мм |
| **70.7Ω** | W70 = 0.95 мм |
| **100Ω** | W100 = 0.55 мм |

---

### 7️⃣ Входное сопротивление (Rin)

**Для питания через вырез (inset feed):**
```
Rin = Rinput × cos²(π × y₀ / 2L)
```

Где:
- `Rinput` ≈ 150-300Ω (край патча)
- `y₀` — глубина выреза от края

**Для 50Ω питания:**
```
y₀ ≈ 1.638 мм (для RO4350B @ 10.5 GHz)
```

---

### 8️⃣ Полоса пропускания (BW)

**Приблизительная формула:**
```
       h     √εeff
BW ≈ ───── × ────── × 100%
       λ₀      εr
```

**Для RO4350B @ 10.5 GHz:**
```
BW ≈ (0.762/28.55) × (√3.08/3.48) × 100% ≈ 2.7%
```

---

### 📊 Сводная таблица для 10.5 GHz (RO4350B)

| Параметр | Формула | Значение |
|----------|---------|----------|
| **Длина волны (λ₀)** | c / f₀ | 28.55 мм |

## 🔌 Правила компоновки для силовых печатных плат (по TI AN-1229 SNVA054C)

### 📋 Основано на:
- **TI AN-1229** SIMPLE SWITCHER® PCB Layout Guidelines (SNVA054C)
- **TI AN-1149** Layout Guidelines for Switching Power Supplies
- **TI Power Management** Thermal & EMI design guidelines

---

### 1️⃣ Размещение компонентов (Component Placement)

#### **Приоритет №1: Входные конденсаторы**
**Правило:** Входной конденсатор должен быть **максимально близко** к выводу VIN микросхемы.

#### **Приоритет №2: Выходной диод (Catch Diode)**
**Правило:** Диод должен быть **максимально близко** к выводу SW микросхемы.

#### **Приоритет №3: Выходные конденсаторы**
**Правило:** Выходные конденсаторы должны быть **близко к выходу** дросселя.

---

### 2️⃣ Земляные плоскости (Ground Planes)

#### **Правила для импульсных БП:**
1. ✅ **Сплошная плоскость** под силовым каскадом
2. ✅ **Много via** для соединения с GND (минимум 4-6 на компонент)
3. ✅ **Термальные via** под мощными компонентами
4. ❌ **Избегать разрывов** в плоскости земли
5. ❌ **Не пропускать** сигнальные трассы через силовую землю

---

### 3️⃣ Силовые дорожки (Power Traces)

#### **Расчёт ширины дорожек (1 oz медь, ΔT=10°C):**

| Ток (A) | Ширина (мм) |
|---------|-------------|
| 1A | 0.3 |
| 2A | 0.6 |
| 3A | 1.0 |
| 5A | 2.0 |
| 10A | 5.0 |

**Правила:**
1. ✅ **Ширина ≥ 0.5мм на 1А** (для 1 oz меди)
2. ✅ **Короткие и широкие** дорожки для силовых цепей
3. ✅ **Углы 45°** или скруглённые (не 90°!)
4. ❌ **Избегать острых углов** (концентрация тока)
5. ❌ **Не использовать** переходные отверстия для токов >2A

---

### 4️⃣ Тепловое управление (Thermal Management)

**Правила:**
1. ✅ **Распределять тепло** по плате равномерно
2. ✅ **Thermal vias** под горячими компонентами (диаметр 0.3мм, шаг 1мм)
3. ✅ **Медные полигоны** для отвода тепла
4. ✅ **Воздушный зазор** ≥5мм от горячих зон
5. ❌ **Не группировать** все горячие компоненты вместе
6. ❌ **Не размещать** термочувствительные компоненты рядом с горячими

---

### 5️⃣ EMI/EMC рекомендации

**Правила:**
1. ✅ **Минимизировать площадь** силовых петель
2. ✅ **Сплошная земля** под силовыми цепями
3. ✅ **Экранирование** чувствительных узлов
4. ✅ **Фильтры** на входе и выходе (LC, RC)
5. ❌ **Избегать длинных параллельных** трасс (перекрёстные помехи)
6. ❌ **Не пропускать** силовые трассы под чувствительными компонентами

---

### 6️⃣ Развязывающие конденсаторы (Decoupling Capacitors)

**Правила:**
1. ✅ **Максимально близко** к выводу питания IC
2. ✅ **Маленький сначала** (0.1µF), потом большой (10µF)
3. ✅ **Короткие выводы** к конденсатору
4. ✅ **Несколько конденсаторов** параллельно (разные номиналы)
5. ❌ **Не размещать** далеко от IC
6. ❌ **Не использовать** длинные трассы к конденсатору

---

### 7️⃣ Трассы обратной связи (Feedback Traces)

**Правила:**
1. ✅ **Короткая трасса** от делителя к FB выводу
2. ✅ **Далеко от силовых** узлов (SW, VIN, VOUT)
3. ✅ **Экранирование** землёй с двух сторон
4. ✅ **Делитель близко** к выводу FB
5. ❌ **Не пропускать** рядом с импульсными узлами
6. ❌ **Не делать длинными** (>20мм)

---

### 8️⃣ Контроль тока (Current Sensing)

**Правила:**
1. ✅ **Кельвинское подключение** (4-проводное)
2. ✅ **Симметричные трассы** от шунта к усилителю
3. ✅ **Далеко от силовых** помех
4. ✅ **Экранирование** землёй
5. ❌ **Не использовать** асимметричные трассы
6. ❌ **Не пропускать** рядом с импульсными узлами

---

### 9️⃣ Входные/Выходные конденсаторы

**Правила:**
1. ✅ **Параллельно** несколько номиналов (10µF + 0.1µF)
2. ✅ **Максимально близко** к IC
3. ✅ **Низкий ESR** для импульсных токов
4. ✅ **Короткие трассы** к конденсаторам
5. ❌ **Не размещать** далеко от IC
6. ❌ **Не использовать** только один номинал

---

### 🔟 Узел коммутации (Switching Node)

**Правила:**
1. ✅ **Короткая дорожка** от SW к дросселю
2. ✅ **Широкая дорожка** для импульсного тока
3. ✅ **Диод близко** к выводу SW
4. ✅ **Маленькая площадь** петли SW-GND
5. ❌ **Не делать длинной** (>10мм)
6. ❌ **Не пропускать** под чувствительными трассами

---

### 1️⃣1️ Пути высокого тока (High Current Paths)

**Правила:**
1. ✅ **Ширина ≥ 1мм на 2А** тока
2. ✅ **Несколько слоёв** для токов >5A
3. ✅ **Переходные отверстия** ≥0.5мм для VIA
4. ✅ **Много VIA** параллельно для токов >2A
5. ❌ **Не использовать** узкие дорожки для силовых
6. ❌ **Не использовать** одно VIA для токов >2A

---

### 📊 Чеклист для проверки силовой платы

**Перед отправкой на производство:**
- [ ] **Входные конденсаторы** близко к VIN выводу
- [ ] **Выходной диод** близко к SW выводу
- [ ] **Выходные конденсаторы** близко к выходу дросселя
- [ ] **Сплошная земляная плоскость** под силовым каскадом
- [ ] **Ширина дорожек** соответствует току
- [ ] **Thermal vias** под горячими компонентами
- [ ] **Развязывающие конденсаторы** близко к каждому выводу VCC
- [ ] **Трасса обратной связи** короткая и экранированная
- [ ] **Токовый шунт** с кельвинским подключением
- [ ] **EMI фильтры** на входе и выходе
- [ ] **Зазоры** ≥0.5мм для силовых цепей
- [ ] **Термический анализ** выполнен

## 🛠️ Building and Running

### Блок-схема системы:

### 🎯 Основные подсистемы:

#### 1️⃣ **Цифровая обработка (FPGA + MCU)**

| Компонент | Модель | Функция |
|-----------|--------|---------|
| **FPGA** | XC7A50T-2FTG256 | Chirp generation, ADC capture, DSP, USB |
| **MCU** | STM32F746ZGT7 | Control, sequencing, sensors, configuration |
| **USB 3.0** | FT601 | High-speed data transfer to host |

**Потоки данных:**
```
FPGA → DAC (TX chirp)
FPGA ← ADC (RX data)
MCU → AD9523, ADF4382, ADAR1000 (configuration)
MCU ← Sensors (IMU, GPS, BMP, TMP)
```

---

#### 2️⃣ **TX Тракт (Передача)**

```
FPGA → DAC → LPF → MIXER → BPF → RF Switch → Combiner → ADAR1000 → ADTR1107 → PA → Antenna
```

| Этап | Компонент | Описание |
|------|-----------|----------|
| **DAC** | AD9708 | 8-bit 125 MSPS — генерация chirp |
| **LPF** | — | Reconstruction filter |
| **MIXER TX** | LTC5552 | Up-conversion до RF |
| **BPF** | — | Band-pass filter 10.5 GHz |
| **RF Switch** | M3SWA2-34DR+ | TX/RX переключение |
| **Combiner** | EP4RKU+ | Деление на 4 канала |
| **Beamformer** | 4× ADAR1000 | 4 канала × 4 элемента = 16 |
| **Front-end** | 16× ADTR1107 | RX LNA / TX PA + switch |
| **PA** | 16× QPA2962 | 10W GaN (только Extended) |
| **Circulator** | — | Изоляция TX/RX |
| **Antenna** | 16×16 / 32×16 | Phased array / waveguide |

---

#### 3️⃣ **RX Тракт (Приём)**

```
Antenna → Circulator → ADTR1107 → ADAR1000 → Combiner → RF Switch → BPF → MIXER → LPF → Opamp → ADC
```

| Этап | Компонент | Описание |
|------|-----------|----------|
| **Front-end** | 16× ADTR1107 | RX LNA (NF ~3-5 dB) |
| **Beamformer** | 4× ADAR1000 | Phase/amplitude control |
| **Combiner** | EP4RKU+ | Суммирование 4 каналов |
| **RF Switch** | M3SWA2-34DR+ | TX/RX переключение |
| **BPF** | — | Band-pass filter 10.5 GHz |
| **MIXER RX** | LTC5552 | Down-conversion до IF |
| **LPF** | — | Anti-aliasing filter |
| **Opamp** | AD8352 | IF amplifier/ADC driver |
| **ADC** | AD9484 | 8-bit 500 MSPS |

---

#### 4️⃣ **Синтез частот (Clock & LO)**

```
OCXO/XO/VCXO → AD9523 → ADF4382 (TX LO) → Mixer
                         → ADF4382 (RX LO) → Mixer
                         → DAC clock
                         → ADC clock
                         → FPGA clock
```

| Компонент | Модель | Частота | Назначение |
|-----------|--------|---------|------------|
| **OCXO** | ECOC-2522 | 10 MHz | Master reference (стабильность) |
| **XO** | CCHD-957 | 100 MHz | System clock |
| **VCXO** | CVHD-950 | 100 MHz | PLL reference |
| **Clock Synth** | AD9523 | — | Low-jitter clock distribution |
| **LO TX** | ADF4382 | 10.5 GHz | TX mixer LO |
| **LO RX** | ADF4382 | 10.5 GHz ± IF | RX mixer LO |
| **Balun** | MTX2-143+ | — | Differential conversion |
| **Atten PAD** | ATS1005 | -3 dB | Level adjustment |

**Выходы AD9523:**
- **CMOS 120 MHz** → DAC
- **LVDS 400 MHz** → ADC
- **CMOS 100 MHz** → FPGA (system/reference)

---

#### 5️⃣ **Датчики и периферия (MCU)**

| Датчик | Модель | Интерфейс | Назначение |
|--------|--------|-----------|------------|
| **IMU** | GY-85 | I2C/SPI | Orientation, acceleration |
| **Barometer** | BMP180 | I2C | Altitude, pressure |
| **GPS** | NEO-6M | UART | Position, timing |
| **Temp Sensor** | TMP37 | ADC | Thermal monitoring |
| **Current Sensor** | INA241A3 | ADC | PA current monitor (×16) |
| **Stepper Driver** | TBS6600 | Step/Dir | Antenna positioner (opt) |
| **Cooling** | — | PWM | Fan control |
| **Relay** | — | GPIO | PA power control |

---

#### 6️⃣ **Питание (Power Distribution)**

**Коннекторы:**

| Коннектор | Назначение | Напряжения |
|-----------|------------|------------|
| **7×2** | To Microcontroller | 3.3V, GND, signals |
| **2×(2×1)** | To LOs | +5V0, +3V3_LO, GND |
| **2×(2×1)** | To Clock Synth | +1V8_CLOCK, +3V3_CLOCK, GND |
| **2×1** | To XOs | +3V3_XO, GND |
| **6×2** | To Microcontroller | Control signals |

**Контроль питания PA:**
```
DAC (AD9708) → Opamp (OPA4703) → Vg (Gate voltage)
ADC (ADS7830) → Idq monitor (current sense)
INA241A3 → Per-channel current (×16)
Relay → VD (Drain power) on/off
```

### 🎯 Конфигурации:

#### **AERIS-10N (Nexus):**
- **Антенна:** 16×16 Phased Array (Patch)
- **Каналов:** 16 (4× ADAR1000 + 16× ADTR1107)
- **PA:** Внутренний PA ADTR1107 (~1W)
- **Дальность:** до 3 км

#### **AERIS-10E/X (Extended):**
- **Антенна:** 32×16 Dielectric Filled Waveguide
- **Каналов:** 16 (с внешними PA)
- **PA:** 16× QPA2962 GaN (10W каждый)
- **Дальность:** до 20 км
- **Охлаждение:** Принудительное (fans + heatsink)

---

### 📝 Примечания к архитектуре:

1. **TX/RX Switching:** M3SWA2-34DR+ обеспечивает изоляцию ~40 dB
2. **Beamforming:** ADAR1000 × 4 = 16 независимых каналов
3. **LO Architecture:** 2× ADF4382 для независимого TX/RX LO
4. **Clock Tree:** AD9523 раздаёт phase-aligned clock
5. **PA Control:** Gate bias через DAC, drain через relay
6. **Current Monitor:** Per-channel sensing для защиты PA
7. **Sensors:** IMU/GPS/Baro для motion compensation (opt)

## 📐 Development Conventions

### RF-требования к антеннам:
1. **Все пути равны**: Одинаковая электрическая длина до всех патчей
2. **Плавные углы**: 45° фаски или дуги (R=1.5-2.5мм)
3. **Рассчитанные ширины**: W50/W70/W100 для RO4350B 0.762мм
4. **T1 посередине**: Делитель T1 точно между P2-P3 (Y=0.0)
5. **T2/T3 симметрично**: T2 между P1-P2, T3 между P3-P4
6. **Via stitching**: Локальные GND via у T1/T2/T3
7. **Сплошной ground**: B.Cu без разрывов

### Именование файлов:
- `AERIS_10N_Patch_4x4_v{версия}.kicad_pcb`
- `AERIS_10N_Patch_Array_8x16_v{версия}.kicad_pcb`

### Версионирование:
- **v1-v20**: Промежуточные версии
- **v21+**: Финальные рабочие версии
- **Чётные**: RF-оптимизированные
- **Нечётные**: Базовые архитектуры
